CryptoLens Overview

CryptoLens is a Smart Money Concepts (SMC) pattern detection platform for cryptocurrency trading. It automatically identifies institutional trading patterns across multiple timeframes and sends push notifications when high-probability setups occur.

Key Features

  • Pattern Detection - Fair Value Gaps, Order Blocks, and Liquidity Sweeps
  • Multi-Timeframe Analysis - 1m, 5m, 15m, 1h, 4h, 1d timeframes
  • Confluence Signals - Higher reliability when patterns align across timeframes
  • Push Notifications - Real-time alerts via NTFY.sh
  • Portfolio Management - Track trades and journal entries
  • Backtesting - Test strategies on historical data
  • 3-Tier Subscriptions - Free, Pro, and Premium plans

Trading Pattern Types

CryptoLens detects three Smart Money Concepts (SMC) patterns that indicate institutional activity. These patterns represent areas where large players have placed orders or hunted liquidity.

Fair Value Gap (FVG) F

A Fair Value Gap is a 3-candle pattern where aggressive buying or selling creates a gap between candle 1's wick and candle 3's wick. This gap represents unfilled orders where price moved too fast.

Detection Logic

  • Bullish FVG: Candle 1 high < Candle 3 low (gap up)
  • Bearish FVG: Candle 1 low > Candle 3 high (gap down)
  • Minimum gap size: Configurable (default 0.1% of price)
Bullish FVG: +-----+ | | <- Candle 3 +-----+ | ==========|=====|=====|====== <- FVG Zone (entry area) | +-----+ +-----+ <- Candle 1 | | +-----+ Bearish FVG: +-----+ | | <- Candle 1 | +-----+ ====|=====|=====|============ <- FVG Zone (entry area) +-----+ | | | <- Candle 3 +-----+

How to Trade

  1. Wait for price to retrace back into the FVG zone
  2. Entry: At the zone edge (conservative) or midpoint (aggressive)
  3. Stop Loss: Beyond the opposite edge of the zone + buffer
  4. Take Profit: 1:2 or 1:3 risk-reward ratio
Pro Tip: FVGs on higher timeframes (4h, 1d) are more significant than lower timeframes.

Order Block (OB) O

An Order Block is the last opposing candle before a strong impulsive move. It represents where institutions placed large orders that caused the subsequent move.

Detection Logic

  • Bullish OB: Last red (bearish) candle before a strong green impulse
  • Bearish OB: Last green (bullish) candle before a strong red impulse
  • Impulse must be > 1.5x average body size
  • Impulse should break recent structure (swing high/low)
Bullish Order Block: +-----+ | ^^^ | <- Strong impulse up | ^^^ | =====|=====|===== <- Order Block zone | v | <- Last red candle (OB) +-----+ Bearish Order Block: +-----+ | ^ | <- Last green candle (OB) =====|=====|===== <- Order Block zone | vvv | | vvv | <- Strong impulse down +-----+

How to Trade

  1. Mark the order block candle's body (open to close)
  2. Wait for price to return to this zone
  3. Entry: Within the OB body
  4. Stop Loss: Beyond the OB wick
  5. Take Profit: Previous swing high/low or 1:2 RR

Liquidity Sweep (Sweep) S

A Liquidity Sweep occurs when price takes out obvious highs/lows (where stop losses cluster) before reversing. Institutions hunt these liquidity pools to fill large orders.

Detection Logic

  • Identify swing highs/lows (using lookback period)
  • Bullish Sweep: Price wicks below swing low, closes back above
  • Bearish Sweep: Price wicks above swing high, closes back below
  • The sweep should be followed by reversal momentum
Bearish Liquidity Sweep: ------------ Previous High (liquidity) | =====|===== <- Sweep wick (takes out stops) | -----+----- <- Close back inside | v Reversal

How to Trade

  1. Identify obvious swing highs/lows where stops cluster
  2. Wait for a sweep (wick beyond, close inside)
  3. Entry: After the sweep candle closes
  4. Stop Loss: Beyond the sweep wick
  5. Take Profit: Opposite side of the range

Signals & Confluence

What is Confluence?

Confluence occurs when multiple timeframes show patterns in the same direction. Higher confluence scores indicate more reliable trade setups.

Confluence Scoring

ScoreMeaningReliability
2 TFsModerate signalStandard
3 TFsStrong signalHigh
4+ TFsVery strong signalVery High
HTF + LTFMulti-timeframe alignmentHighest

Signal Generation

Signals are generated when:

  1. At least 2 timeframes show patterns in the same direction
  2. Patterns are from the same symbol
  3. Confluence score meets the user's minimum threshold

Signal Contents

Each signal includes:

  • Symbol - Trading pair (e.g., BTC/USDT)
  • Direction - LONG or SHORT
  • Entry Price - Suggested entry level
  • Stop Loss - Risk management level
  • Take Profit 1 & 2 - Profit targets
  • Confluence Score - Number of aligned timeframes
  • Timeframes - Which TFs are aligned
Important: Signals are for informational purposes only. Always do your own analysis and manage risk appropriately.

Backtesting

The backtesting module allows you to test pattern-based strategies on historical data. This helps evaluate strategy performance before trading with real capital.

Backtest Parameters

ParameterDescription
SymbolTrading pair to test
TimeframeChart timeframe
Start DateBacktest start date
End DateBacktest end date
Pattern TypeFVG, Order Block, or Liquidity Sweep
R:R TargetRisk-to-reward ratio (e.g., 2.0 = 1:2)

Backtest Results

  • Total Trades - Number of trades taken
  • Win Rate - Percentage of winning trades
  • Average R:R - Average risk-reward achieved
  • Total Profit % - Cumulative profit percentage
  • Max Drawdown - Largest peak-to-trough decline
  • Trade List - Individual trade details

Trade Simulation Logic

  1. Pattern detected at candle N
  2. Wait for price to enter the zone (entry trigger)
  3. If entry triggered, monitor for SL or TP hit
  4. Record result: Win (TP hit), Loss (SL hit), or No Fill
Note: Backtesting is available for Premium tier users only.

Deployment

Production Checklist

  1. Set FLASK_ENV=production
  2. Use PostgreSQL instead of SQLite
  3. Configure HTTPS with valid SSL certificate
  4. Set up Redis for rate limiting and caching
  5. Configure reverse proxy (nginx)
  6. Set up cron jobs for background tasks
  7. Configure log aggregation
  8. Set up monitoring (Prometheus + Grafana)

Security

Authentication Security

  • Password Hashing - Werkzeug PBKDF2-SHA256
  • 2FA Support - TOTP-based two-factor authentication
  • Rate Limiting - Prevents brute force attacks
  • Account Lockout - After 5 failed login attempts
  • Session Security - Secure, HTTP-only cookies

CSRF Protection

All forms include CSRF tokens via Flask-WTF. API endpoints are exempt but require API key authentication.

API Security

  • API Keys - SHA-256 hashed, timing-safe comparison
  • Rate Limiting - Per-endpoint limits
  • Tier Restrictions - Premium-only API access

Payment Security

  • Webhook Verification - HMAC signature validation
  • No Card Storage - All payment data handled by providers
  • PCI Compliant - Via LemonSqueezy integration
Security Reminder: Change default admin password immediately after setup. Never commit .env files or API keys to version control.
